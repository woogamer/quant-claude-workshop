# 발표자 cheat sheet — 6/4 워크샵

발표·시연 중 옆에 띄워놓고 보는 자료. 명령어 + 예상 응답 + 막혔을 때 폴백.

---

## 발표 5분 전 점검

- [ ] 노트북에서 Codespace 열려 있고 `claude` 실행 가능한 상태
- [ ] `config.yaml` KIS 키 채워져 있음 (`./setup.sh` 완료)
- [ ] `python scripts/demo_price.py 005930` 한 번 돌려 KIS 연결 확인
- [ ] `claude` 한 번 실행해 OAuth 토큰 유효한지 확인 (안 그러면 시연 중 로그인 창 뜸)
- [ ] **사전 매수 (선택)**: 어제 22:30 이후 본인 모의계좌로 NVDA 1주 미리 매수 → 시연 중 잔고 조회할 거리 확보
- [ ] 발표 화면 공유: VS Code 폰트 16pt 이상으로

## 시간대 현실

| 시간 | 한국주식 | 미국주식 |
|---|---|---|
| **워크샵 19:00~20:00** | 장 마감 (15:30 종료) | 장 시작 전 (22:30 시작) |
| 22:30 이후 | (다음 영업일) | 정규장 |

→ **시연 시간엔 한국·미국 둘 다 매수 거절될 가능성**. 아래 폴백 패턴으로 진행.

## 시연 3개 — 명령어 + 예상 응답

### 시나리오 1 — 단순 매수 (2분)

**입력**: `> 엔비디아 현재가 알려줘`

**예상**: Claude 가 `get_overseas_stock_price("NVDA", "NAS")` 호출 →
```json
{"심볼": "NVDA", "현재가_USD": "...", "등락률": "..."}
```

**입력**: `> 1주 매수해줘`

**라이브 OK 시 응답**: `buy_overseas_stock` 호출 → "미국주식 매수 주문 완료"

**라이브 거절 시 (장 시작 전)**:
```
응답: 모의투자 장시작전 입니다.
```
→ **그대로 보여주고 설명**: "지금 한국 시간 19시는 미국장 시작 전이에요. 22:30 이후엔 그대로 체결됩니다. 어제 22:30에 미리 한 번 매수해뒀는데, 잔고를 보면..."

**폴백 명령**: `> 내 미국주식 잔고 보여줘` → 어제 매수해둔 NVDA 1주 표시

### 시나리오 2 — 알고리즘 신호 + 매수 (2분)

**입력**: `> 엔비디아 골든크로스 신호 어때?`

**예상**: `golden_cross_signal("NVDA")` 호출 →
```json
{"market": "US", "signal": "HOLD" 또는 "GOLDEN_CROSS_BUY",
 "ma5": ..., "ma20": ..., "interpretation": "..."}
```

신호가 GOLDEN_CROSS_BUY 면 매수 권유, HOLD 면 다른 종목 시도.

**입력**: `> 그럼 삼성전자도 봐줘`

→ `golden_cross_signal("005930")` → `market: "KR"` 자동 판별 보여줌.

**입력**: `> 매수 시그널이 가장 강한 종목 1주 사줘`

→ Claude가 자체 판단으로 종목 선택. 매수 시도 → 시간 외라 거절 → 폴백 멘트.

### 시나리오 3 — 공시 기반 매수 (3분)

**입력**: `> DART 공시 찾아봐서 내부자 매수 시그널 있는 종목 알려줘`

**예상**: `insider_buying_signals(days_back=5, limit=10)` 호출 →
```json
{
  "count": 8,
  "signals": [
    {"corp_name": "에스테크엠", "net_shares_acquired": 202500,
     "net_rate_change_pct": 13.4, ...},
    ...
  ]
}
```

→ Claude 가 가장 강한 시그널 1~2개 추천 + 이유 설명.

**입력**: `> 가장 강한 시그널 1주 매수해줘`

→ 한국 종목이라 한국장 시간 외 거절. **이 시점에 자연스럽게 마무리**:

> "오늘 워크샵 시간엔 두 시장 다 매수가 안 되지만, 22:30 이후 본인 환경에서 미국 종목으로 똑같이 해보시면 그대로 체결됩니다. 다음 30분 실습에서 직접 띄워봅시다."

## 실습 30분 — 자주 나올 질문

### Q. KIS 키가 401 에러 나요
- A. config.yaml 열어서 App Key·Secret 직접 확인. 가장 흔한 건 **공백 포함** 또는 **줄바꿈 포함**. 트리밍하고 재실행.

### Q. Claude OAuth 로그인 창이 안 떠요
- A. VS Code 하단 "포트(Ports)" 탭 클릭 → 콜백 포트 옆 지구본 아이콘 클릭 → 새 탭에서 로그인.

### Q. Claude 가 buy 도구를 안 불러요
- A. `claude` 종료(Ctrl+C·Ctrl+D) → 재실행. `.mcp.json` 변경 후엔 항상 재실행 필요.

### Q. 시장 시간 외라 매수가 안 되는데 다음 영업일 예약은 안 되나요?
- A. KIS 모의투자는 예약주문 미지원. 22:30 이후 미국 종목으로 다시 시도하거나, 다음 날 09:00 이후 한국 종목.

### Q. 코드 자체를 수정하고 싶어요 (예: 골든크로스 5/20 → 10/30)
- A. Claude 에게 그대로 자연어로 시키세요: `> scripts/golden_cross.py 에서 5/20을 10/30으로 바꿔줘` → Claude 가 Edit 도구로 직접 수정.

### Q. Codespace 60시간 한도 다 쓰면?
- A. 평소 자기 노트북에 Python·Claude Code 설치해도 동일하게 작동. 또는 Codespace 사용 후 즉시 중지.

## 발표 중 시간 관리

| 슬라이드 | 누적 시간 | 멘트 핵심 |
|---|---|---|
| 1 · 제목 | 0:00 | 인사 + 한 줄 소개 |
| 2 · 오늘 1시간 | 0:30 | 발표 15·실습 30·QnA 15 |
| 3 · 6개 전략 표 | 1:30 | **가동 2 / 페이퍼 3 / 멈춤 1**. 데이터로 결정 |
| 4 · 삽질 4가지 | 3:00 | 5/7 사고 · 넛지 증폭 · Toss 발굴 · B 자본 부족 |
| 5 · 지금 결과 (5/21) | 5:00 | A·B 잔고 + PRISM-C 페이퍼 — **결합 가설 약화** |
| 6 · 시스템 전체 그림 | 6:30 | 큰 그림. 사람 손 없이 돈다 |
| 7 · Claude 가 어디 들어가나 | 7:30 | **자연어 → 도구 호출** 패턴 강조 |
| 8 · MCP 도구 9개 | 8:30 | 매매 + 신호. 이제 보여드림 |
| 9 · 라이브 시연 시작 | 9:00 | 화면 전환. **여기서 6분** |
| 10 · 자동 vs 사용자 확인 | 15:00 | 매수는 항상 확인 |
| 11 · 실습 안내 | 16:00 | 다음 30분 흐름 |

→ 슬라이드 4·5 가 새로 추가된 부분. 시연 시작 직전(8:30)까지 시계 보기. **시연을 7→6분으로 단축** (슬라이드 늘었음). 넘어가면 시나리오 3 생략.
