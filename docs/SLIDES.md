## 지난 2달 — 무엇을 시도했나

**목표**: 5천만 원 × 5개 전략 = 5개 독립 전략 운영

**결과**: 자동매매로 살아남은 전략 — **2개**

- **A. 컨센서스 포트폴리오** — 액티브 ETF 35개가 동시에 담는 종목 = 시장 컨센서스
- **B. PRISM 리서치** — Claude 가 매주 후보 종목을 5단계로 깊게 분석

나머지 3개? **설계와 검증에서 멈춤**. 한 전략 검증에 백테스트·데이터 정합성 확인으로 **몇 주씩** 걸림.

---

## 솔직히 — 1인 파트타임의 한계

```
학교일  →  회사일  →  동아리
```

이 우선순위에서 동아리는 자주 밀렸습니다.

> 의미있는 수익률을 내려면 노력이 훨씬 더 필요하다는 걸
> **2달 동안 데이터로 직접 확인했습니다.**

그래도 가장 크게 얻은 건 단순한 "수익 전략" 하나가 아니라 —

**AI 도구를 실제 주식 매매 환경에 붙여본 경험** 그 자체였습니다.

---

## 자동매매 시스템 전체 그림

```
ETF 35개 스크래핑 (주 1회)
        ↓
  컨센서스 스코어링
   (시간감쇠 + AUM + 중복보정 + DART 가산)
        ↓
   Top 20 후보 → PRISM 5-Layer 리서치
                  (매주 2회 자동)
        ↓
   포트폴리오 가중치 산출
        ↓
   KIS API 자동매매 (cron 17개)
        ↓
   텔레그램 일일 리포트 (오전 8시)
```

매일 / 매주 사람 손 없이 돕니다. 모의계좌 2개 (Budget A · B) + 페이퍼 1개 (C).

---

## 핵심 차별점 — Claude Code 가 어디에 들어가는가

기존 방식:
> 사람이 코드 짜고 → 사람이 실행 → 사람이 결과 해석

이번 시스템:
> 사람이 자연어로 "삼성전자 매수해줘"
>   → **Claude 가 도구를 골라 호출**
>     → KIS API → 모의계좌 체결

도구 9개를 미리 만들어두고, **Claude 는 사용자 의도만 보고 어떤 도구를 어떻게 호출할지 결정**합니다.

매매 / 잔고 / 신호 분석 / 공시 조회 — 모두 같은 패턴.

---

## 도구 (MCP) 9개

**매매 — kis-broker**
- get_stock_price · get_balance · buy_stock · sell_stock (한국)
- get_overseas_stock_price · get_overseas_balance · buy_overseas_stock · sell_overseas_stock (미국)

**신호 — signals**
- golden_cross_signal — 5/20 이동평균 (한국·미국 자동 판별)
- dart_recent_disclosures — 최근 N일 지분공시
- insider_buying_signals — 임원·주요주주 순매수만 추출

워크샵 키트 (`quant-claude-workshop`) 에 다 들어있습니다. Codespace 띄우면 자동 로드.

## 라이브 시연 — 자연어로 매수까지

(화면 전환 · 약 7분)

### 시나리오 1 — 단순 매수
```
> 엔비디아 현재가 알려줘
> 1주 매수해줘
```

### 시나리오 2 — 알고리즘 신호 + 매수
```
> 엔비디아 골든크로스 신호 어때?
> 신호 좋으면 1주 매수해줘
```

### 시나리오 3 — 공시 기반 매수
```
> DART 공시 찾아봐서 내부자 매수 시그널 있는 종목 알려줘
> 가장 강한 시그널 1주 매수해줘
```

## 그래서 어디까지 자동인가

| 단계 | 자동? |
|---|---|
| 종목 발굴 (스크래핑·컨센서스) | ✅ 매주 자동 |
| 후보 깊은 리서치 (PRISM 5-Layer) | ✅ 주 2회 자동 (Claude 헤드리스) |
| 매수 주문 | ⚠️ **사용자 확인 후** 자동 |
| 일일 리포트 | ✅ 매일 텔레그램 자동 |

**원칙**: 의사결정 (매수/매도) 은 항상 사용자 확인. Claude 가 후보까지만 제시.
