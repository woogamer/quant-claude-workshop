→ "5개 다 돌리겠다"는 처음 목표였고, 검증·실패·재설계 거치면서 위 표가 남았습니다.

## 삽질 4가지 — 인프라가 늘어난 이유

**① 5/7 broken portfolio 사고**
PRISM 06:00 cron 도중 Claude CLI 자가 업데이트 윈도우 → 18/35 호출 실패 → fallback portfolio 가 자동매매됨. **점심까지 모름.**
→ retry 3회·sanity gate·`.FAILED.json` 분리 4개 안전장치 추가 (5/8)

**② Claude 넛지 증폭 (4/24)**
같은 데이터로 Claude 넛지 버전 vs Unbiased 버전 A/B → 삼성전자 **-1.5% vs +1.0%** 반대 방향. 내가 매크로 의견 한 줄 더 던지면 LLM이 그걸 캡으로 증폭.
→ 사용자 넛지 → 프롬프트 방향성 확장 금지 룰

**③ Toss 인기순위 endpoint Playwright 발굴 (5/5)**
단순 fetch 차단 → chunk 분석도 막힘 → **Playwright 자동화로 ajax endpoint 캡처** → header 1개로 fetch 성공.
→ PRISM-C 의 핵심 알파 소스

**④ Budget B 1천만의 한계 (5/18)**
target 76% vs 실제 55%, **gap 3.46m**. 단가 60만+ 종목 1주도 못 사서 6종목 누락. LLM 의도는 명확한데 매매 봇이 구조적으로 못 산 케이스.

## 자동매매 시스템 전체 그림

```
ETF 35개 스크래핑 (주 1회)
        ↓
  컨센서스 스코어링
   (시간감쇠 + AUM + 중복보정 + DART 가산)
        ↓
   Top 20 후보 → PRISM 5-Layer 리서치
                  (매주 2회 자동)
        ↓
   포트폴리오 가중치 산출
        ↓
   KIS API 자동매매 (cron 17개)
        ↓
   텔레그램 일일 리포트 (오전 8시)
```

매일 / 매주 사람 손 없이 돕니다. 모의계좌 2개 (Budget A · B) + 페이퍼 1개 (C).
